

## 1. 기획 의도 (왜 필요한가?)

- 플레이어의 생존 여부를 직관적인 수치(정수)로 관리하여 긴장감을 부여함.
    
- 사망 시 명확한 피드백과 함께 게임 흐름을 차단하여 재도전을 유도함.
    

## 2. 유저 플로우 (플레이어가 뭘 하는가?)

1. $$피격$$
    
    적의 공격이나 투사체에 닿으면 체력이 감소합니다.
    
    - 피해량 계산: (적 공격력 - 내 피해 감소 수치) 만큼 차감.
        
2. $$반응 - 무적$$
    
    피격 즉시 캐릭터가 잠시 무적 상태가 되며 깜빡거립니다. (연속 피격 방지)
    
3. $$위기 알림$$
    
    체력이 **30% 미만**으로 떨어지면 체력 게이지나 화면 가장자리가 붉게 점멸합니다.
    
4. $$회복$$
    
    회복 아이템 획득 시 체력이 즉시 차오릅니다. (최대 체력 초과 불가)
    
5. $$사망/게임오버$$
    
    체력이 0 이하가 되면 캐릭터가 쓰러지고 조작이 불가능해집니다.
    
    - 이후 결과 화면(재시작/로비 버튼)이 출력됩니다.
        

## 3. 수치 / 데이터

| **항목** | **값** | **비고** |

| **기본 최대 체력** | **100** (예시) | 기획서상 변수 'x' |

| **피격 무적 시간** | **1.0초** (예시) | 기획서상 변수 'z' |

| **위기 알림 기준** | **30%** | 전체 체력 대비 비율 |

| **사망 판정 값** | **<= 0** | 0 이하 도달 시 즉시 사망 |

> 💡 모든 체력 수치는 **정수(Integer)** 단위로 관리해주세요.

## 4. 예외 상황 (~하면 어떻게 되나?)

- $$무적 중 피격$$
    
    무적 시간(z초) 동안 또 맞으면? → **피해 없음** (데미지 0 처리).
    
- $$회복 초과$$
    
    체력이 90/100인데 20짜리 힐을 먹으면? → **100으로 보정** (최대 체력 초과 불가).
    
- $$사망 후 피격$$
    
    이미 HP가 0이 되어 쓰러지는 모션 중에 또 맞으면? → **무시** (추가 타격 없음).
    
- $$동시 사망$$
    
    적과 동시에 죽으면? → **패배(게임오버)** 처리.
    

## 5. 연관 시스템

- **HUD UI:** 체력바 표시 (현재/최대 비율), 30% 미만 점멸 이펙트.
    
- **전투 시스템:** 피격 판정 및 데미지 공식 적용.
    
- **게임 루프 매니저:** 게임 오버 시 타임 스케일 정지 혹은 조작 차단 후 UI 팝업 호출.
    

## 6. 레퍼런스 (선택)

- 일반적인 액션 RPG의 체력 및 무적 시스템 표준을 따름.
