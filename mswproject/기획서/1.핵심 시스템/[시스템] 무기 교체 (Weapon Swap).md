## 1. 기획 의도 (왜 필요한가?)

- 캐릭터 하나로 모든 상황을 해결하지 못하게 하고, 상황에 맞춰 4가지 무기를 전략적으로 교체하며 싸우도록 유도함.
    
- 무기 교체 시 시간을 멈추어(일시정지), 피지컬보다는 전략적 선택을 강조함.
## 2. 유저 플로우 (플레이어가 뭘 하는가?)

1. 플레이어가 인게임에서 **[F] 키**를 누른다.
    
2. 게임이 즉시 **일시정지(Pause)** 되며, 캐릭터의 이동 및 공격 입력이 차단된다.
    
3. 화면 중앙에 **4방향 무기 선택 UI(Radial Menu)**가 팝업된다.
    
4. 마우스 커서 또는 WASD 키를 조작하여 원하는 무기 슬롯을 선택(하이라이트)한다.
    
5. 선택 확정(클릭 또는 키 떼기 등) 시 UI가 사라지고, 게임이 **재개(Resume)**되며 선택한 무기가 캐릭터 손에 들린다.

## 3. 수치 / 데이터

|            |             |                               |
| ---------- | ----------- | ----------------------------- |
| **항목**     | **값**       | **비고**                        |
| 교체 진입 키    | F           | 변경 가능                         |
| 무기 슬롯 개수   | 4개          | 고정 (획득 방식 아님)                 |
| 교체 중 게임 속도 | 0 (정지)      | UI 출력 중에는 월드 시간 정지            |
| 데이터 저장 단위  | 캐릭터 별 개별 저장 | A캐릭터의 무기 상태와 B캐릭터의 무기 상태는 독립적 |

## 4. 예외 상황 (~하면 어떻게 되나?)

- **무기를 선택하지 않고 취소하면?**
    
    - (제안) F키를 다시 누르거나 ESC를 누르면 교체 없이 UI가 닫히고 게임 재개.
        
- **사망 상태이거나 CC기(기절 등)에 걸렸을 때?**
    
    - 교체 UI 진입 불가.
        
- **A캐릭터에서 1번 무기를 쓰다가 B캐릭터로 태그 후 다시 A로 돌아오면?**
    
    - A캐릭터가 마지막에 들고 있던 무기(1번)를 들고 나와야 함. (캐릭터별 장착 상태 저장)

## 5. 연관 시스템

- **캐릭터 태그 시스템:** 캐릭터 교체 시 무기 데이터(탄약, 장전 상태) 백업 및 로드 필요.
    
- **HUD:** 현재 장착 중인 무기 아이콘 표시.
    
- **인벤토리/장비:** (추후 확장 시) 무기 4종의 스펙 관리.
